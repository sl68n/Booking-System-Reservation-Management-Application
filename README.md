A Booking System application designed to manage reservations efficiently through structured object-oriented programming and modular system design.

---

# 📌 Project Overview

The Booking System allows users to:

- Create reservations
- Manage customer information
- Track bookings
- Update or cancel reservations
- Handle system validation and constraints

The system follows clean OOP principles and structured program flow.

---

# 🧠 System Features

- Add new bookings
- Edit reservation details
- Cancel reservations
- Display booking records
- Validate user input
- Organized data handling

---

# 🏗️ System Design

The system is structured using:

- Object-Oriented Programming (OOP)
- Class-based modeling
- Encapsulation
- Modular separation of logic

Main Components:

- Booking Class
- Customer Class
- System Controller
- Data Handling Module

---

# ⚙️ Functional Requirements

- Accept user input for reservations
- Prevent duplicate or invalid entries
- Store booking data
- Allow searching and updating records
- Maintain consistent system state

---

# 🔒 Non-Functional Requirements

- Usability
- Data integrity
- Reliability
- Structured code readability

---

# 🚀 Concepts Applied

- OOP Design
- Class Modeling
- Input Validation
- Data Structures
- Modular Programming
